About this experiment

In this experiment, I fixed a bug in the code. I obeserved that if refresh is done on the publicly accessable pages (View) when user is logged in , it causes to flush value of rootscope current user. So it used to give a false impression that user is logged out after refresh, although user was still logged in. This is fixed by checking if the user is logged in currently by LoginService. Profile, setting pages aleardy had this check. So newly created service was incorporated in Home, View , Contact, View detail pages.

Code snippet

login Service 

 
    app.factory('LoginService', function LoginService($http) {
	console.log("In LoginService ");
    var chkLogin = function(){
    	
    	$http.get('/rest/loggedin').success(function(user){
    	 
			if (user !== '0'){
	        	return user;
	        }     
        
    	});
    }
	
	return {
        chkLogin: chkLogin 
    }
    });
